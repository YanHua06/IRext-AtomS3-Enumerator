## Project Overview

IRext AtomS3 Enumerator — ESP32-S3 (M5Stack AtomS3 Lite) firmware for offline IR remote code enumeration. Uses the [IRext](https://opensource.irext.net) open-source IR code library to brute-force test IR codes against target appliances without the original remote.

## Build & Flash

```bash
pio run                 # compile
pio run -t upload       # flash firmware
pio run -t uploadfs     # flash LittleFS data partition (IR bin files)
```

PlatformIO CLI (`pio`) is at `C:\Users\qjy20\.platformio\penv\Scripts\pio.exe`.

## Architecture

**Single-file firmware** — `src/main.cpp` (~1260 lines) contains everything: IR send/receive, enumeration state machine, IR learning, Web UI (HTML/CSS/JS as PROGMEM string), WebSocket server, TCP mode-B protocol, serial command parser, and button handler.

**IRext C library** (`src/ir_decode/`) — unmodified upstream. `ir_binary_open()` parses proprietary `.bin` files, `ir_decode()` produces raw IR timing arrays. Called via `extern "C"` from `main.cpp`.

### Two Operating Modes (auto-detected at boot)

| Mode | Trigger | Behavior |
|------|---------|----------|
| A (Standalone) | LittleFS has `.bin` files | Fully offline enumeration, button + Web terminal control |
| B (TCP Terminal) | LittleFS empty | Waits for computer to push bins via TCP (`upload_bins.py remote`) |

### IR Pipeline

1. Load IRext `.bin` from LittleFS → `ir_binary_open(cat, subcat, data, len)`
2. Build AC status struct or pass TV keycode → `ir_decode(funcCode, rawBuf, &ac)` → raw timing array
3. `IrSender.sendRaw(rawBuf, rawLen, 38kHz)`

### IR Output Selection

- Internal: GPIO4 (onboard IR LED)
- External: GPIO6 (external emitter module, 5V powered)
- Switch at runtime via `IrSender.setSendPin()` — works because `IR_SEND_PIN` is NOT defined, so `sendPin` is a mutable member

### IR Learning

- Receiver: GPIO39 (external IR receiver, 3.3V powered)
- Captured raw timings stored as `uint16_t` microsecond arrays in `/learned/ir_NNNN.raw`
- Index: `/learned/index.json` (JSON array)
- 30-second timeout auto-exits learning mode
- `irSend()` stops receiver during send to prevent self-capture

### Web UI

- Single-page terminal emulator served from PROGMEM
- WebSocket at `ws://192.168.4.1/ws` for bidirectional text commands
- AC control panel: power, temp, mode, fan, swing
- IR output selector: internal/external toggle
- Learning panel: start/stop/save/list/play/delete

### Command System

Plain-text commands via WebSocket, Serial (115200 baud), or TCP. All parsed by `handleCommand()`. Terminal output via `termPrint()` which broadcasts to Serial + WebSocket + history buffer.

### Companion Python Script

`upload_bins.py` — two subcommands:
- `prepare`: filter IRext SQLite DB by brand/category, copy matching `.bin` files to `data/` with renamed filenames (`<cat>_<sub>_<seq>.bin`)
- `remote`: push bins one-by-one over TCP (Mode B) with interactive confirmation

## Key Constraints

- LittleFS partition ~1.4MB — limit IR bin files per brand batch
- LittleFS filename limit ~28 bytes — `upload_bins.py prepare` handles truncation, full mapping in `_manifest.txt`
- `IRRawbufType` is `uint8_t` (50µs ticks, max 12.75ms per tick) — learned codes stored as `uint16_t` microseconds to preserve full timing resolution
- `IrSender.sendPin` is mutable only because `IR_SEND_PIN` is NOT in build_flags — do not add it
